--------------------------------------------------

توضیح پروژه
این پروژه به‌عنوان تمرین دوم درس GIS تحت وب انجام شده است.
هدف تمرین: پیاده‌سازی یک برنامه WebGIS سمت کاربر با استفاده از JavaScript و سرویس‌های تحت وب است.

در این پروژه یک نقشه تعاملی نمایش داده می‌شود که کاربر می‌تواند:
- یک مکان را جستجو کند.
- یا با کلیک روی نقشه، اطلاعات هواشناسی آن نقطه را مشاهده کند.

--------------------------------------------------
 خلاصه روند انجام پروژه: 
در این پروژه یک WebGIS ساده با استفاده از کتابخانه OpenLayers پیاده‌سازی شده است که امکان جستجوی مکانی (Geocoding) و نمایش اطلاعات آب‌وهوا بر اساس موقعیت جغرافیایی را فراهم می‌کند.

روند انجام پروژه به‌صورت زیر بوده است:
- ریپازیتوری تکلیف از GitHub Fork شده و پروژه به‌صورت محلی Clone گردیده است.
- ساختار اصلی پروژه شامل فایل‌های HTML، CSS و JavaScript ایجاد شده است.
- نقشه تعاملی با استفاده از OpenLayers راه‌اندازی شده و لایه پایه OpenStreetMap به آن اضافه شده است.
- قابلیت جستجوی مکان (Geocoding) پیاده‌سازی شده تا کاربر بتواند نام یک مکان را وارد کرده و نقشه روی آن مکان زوم شود.
- قابلیت دریافت و نمایش اطلاعات آب‌وهوا با کلیک روی نقشه پیاده‌سازی شده است.
- برای بهبود تجربه کاربری، تغییرات ظاهری (UI/UX) و انیمیشن‌های ساده CSS به پروژه اضافه شده‌اند.
- به‌منظور رعایت اصول امنیتی، کلید API داخل کد اصلی قرار داده نشده و در فایل جداگانه نگهداری شده است.
- تغییرات پروژه به‌صورت مرحله‌ای با git commit ثبت شده و در نهایت به GitHub push شده‌اند.

## دستورالعمل‌های راه‌اندازی پروژه (Setup Instructions)
 پیش‌نیازها:
- مرورگر وب (Chrome یا Firefox)
- VS Code (پیشنهادی)
- افزونه Live Server (اختیاری)

ابزارها و فناوری‌های استفاده‌شده:
- HTML
- CSS
- JavaScript
- OpenLayers
- OpenWeather API

امکانات پروژه:
- نمایش نقشه تعاملی با استفاده از کتابخانه OpenLayers
- جستجوی مکان (Geocoding)
- دریافت مختصات جغرافیایی با کلیک روی نقشه
- نمایش اطلاعات هواشناسی شامل:
  • دما
  • وضعیت هوا
  • رطوبت

## پیکربندی کلیدهای API (API Key Configuration)

برای رعایت امنیت:
- کلید API در کد اصلی پروژه قرار داده نشده است.
- فایل `config.js` فقط به‌صورت محلی استفاده می‌شود و در GitHub commit نمی‌شود.
- فایل `config.example.js` به‌عنوان نمونه ساختار کلید API در ریپازیتوری قرار داده شده است.

 اجرای پروژه
1. پوشه پروژه را در VS Code باز کنید.
2. فایل `index.html` را با Live Server اجرا کنید  
   یا  
3. پروژه را از طریق یک سرور محلی ساده اجرا نمایید.

مقایسه APIهای هواشناسی

OpenWeather API
- نسخه رایگان: دارد
- نیاز به ثبت‌نام: دارد
- سادگی استفاده: بالا
- کیفیت مستندات: کامل و واضح
- پشتیبانی از مختصات جغرافیایی: دارد
- مناسب برای پروژه دانشجویی: بسیار مناسب

WeatherAPI
- نسخه رایگان: دارد (با محدودیت بیشتر)
- نیاز به ثبت‌نام: دارد
- سادگی استفاده: متوسط
- کیفیت مستندات: خوب
- پشتیبانی از مختصات جغرافیایی: دارد
- مناسب برای پروژه دانشجویی: نسبتاً مناسب

Meteostat
- نسخه رایگان: دارد
- نیاز به ثبت‌نام: ندارد
- سادگی استفاده: پایین
- کیفیت مستندات: فنی و پیچیده
- پشتیبانی از مختصات جغرافیایی: دارد
- مناسب برای پروژه دانشجویی: کمتر مناسب

نتیجه مقایسه و انتخاب API
با توجه به مقایسه انجام‌شده، OpenWeather API به دلیل داشتن نسخه رایگان مناسب،
سادگی استفاده، مستندات کامل و سازگاری بالا با پروژه‌های WebGIS دانشجویی،
به‌عنوان API نهایی این پروژه انتخاب شد.

--------------------------------------------------
مقایسه APIهای ژئوکدینگ (Geocoding APIs)

Nominatim (OpenStreetMap)
- نسخه رایگان: دارد
- نیاز به ثبت‌نام: ندارد
- نیاز به API Key: ندارد
- کاربرد اصلی: پروژه‌های آموزشی و غیرتجاری
- دقت: متوسط (کافی برای کاربردهای آموزشی)
- محدودیت استفاده: دارد
- هزینه: رایگان

Google Geocoding API
- نسخه رایگان: دارد (محدود)
- نیاز به ثبت‌نام: دارد
- نیاز به API Key: دارد
- نیاز به فعال‌سازی Billing: دارد
- کاربرد اصلی: پروژه‌های تجاری و حرفه‌ای
- دقت: بسیار بالا
- هزینه: پرداخت به‌ازای هر درخواست

Mapbox Geocoding API
- نسخه رایگان: دارد (محدود)
- نیاز به ثبت‌نام: دارد
- نیاز به API Key: دارد
- کاربرد اصلی: اپلیکیشن‌های نقشه‌محور حرفه‌ای
- دقت: بالا
- محدودیت درخواست ماهانه: دارد
- هزینه: پلن پولی پس از Free Tier

HERE Geocoding API
- نسخه رایگان: دارد (محدود)
- نیاز به ثبت‌نام: دارد
- نیاز به API Key: دارد
- کاربرد اصلی: سیستم‌های ناوبری و حمل‌ونقل
- دقت: بالا
- محدودیت درخواست: دارد
- هزینه: پلن پولی
